    express(options)

    Express middleware

    This middleware will create application instance (or get existing by name) on Bandwidth server and configure callback routes. It adds to request instance next keys:

    • bandwidthApi - Bandwidth API instance,
    • applicationId - Application Id on Bandwidth server,
    • phoneNumber - reserved phone number which events will be handled by this application (only if options.phoneNumber is defined),
    • domainId - SIP domain Id (only if options.sip.domain is defined),
    • getOrCreateEndpoint(sipAccount) - function to get sip endpoint data by sip account name (it will create new account if it is absent, only if options.sip.domain is defined).

    apiToken string

    API secret
    phoneNumber object <optional>

    Options for reserving phone number. If it is missing no phone number will be reserved.

    Name Type Attributes Description
    name string <optional>

    Name of reserved phone number
    phoneType string <optional>

    'local' or 'tollFree'
    * string <optional>

    Other options will be passed directly to POST /availableNumbers/local or /availableNumbers/tollFree (see for more details http://dev.bandwidth.com/ap-docs/methods/availableNumbers/postAvailableNumbersLocal.html)
    sip object <optional>

    Sip domain options

    Name Type Attributes Description
    domain string <optional>

    Sip domain name to use (not more than 16 low cased symbols )
    callCallback function | object <optional>

    Function like async function(eventData, req){} or object like {'eventName': async function eventHandler(eventData, req){}} to handle calls callback events
    messageCallback function | object <optional>

    Function like async function(eventData, req){} or object like {'eventName': async function eventHandler(eventData, req){}} to handle message callback events
    cache object <optional>

    Cache manager options like {store: RedisStore}. More details here https://github.com/BryanDonovan/node-cache-manager

    // Samples of usage of req.getOrCreateEndpoint:
req.getOrCreateEndpoint('sipUser1').then(endpoint => {}); // if enpoint sipUser1 is missing new enpoint will be created with random password
req.getOrCreateEndpoint({name: 'sipUser1' password: '123456'}).then(endpoint => {}); // if enpoint sipUser2 is missing new enpoint will be created with given password
    app.use(middlewares.express({
	name: 'My app',
	auth: {userId: 'bandwidthUserId', apiToken: 'bandwidthApiToken', apiSecret: 'bandwidthSecret'},
	phoneNumber: {
		phoneType: 'local',
		areaCode: '910'
	},
	callCallback: async (data, ctx) => {
		// Handle calls events here
		if(data.eventType === 'answer' && ctx.phoneNumber === data.to){
			console.log('Answered');
		}
	}
}));

app.use(middlewares.koa({

	phoneNumber: {

		state: 'NC',
		city: 'Cary'
	},
	callCallback: {

}));
